# Piste - パーソナルトレーニング管理アプリ 計画書

## 1. プロジェクト概要

**Piste** は、パーソナルトレーニングジムの業務を一元管理するWebアプリ。
入会 → カウンセリング → 日々のトレーニング → AI分析 の一連のフローをデジタル化する。
会員はLINEリッチメニューからアクセスし、PWAとしてスマホで利用する。

## 2. ターゲットユーザーとアクセス経路

- **会員 (ユーザー)**: LINE公式アカウントのリッチメニュー (既存) の1セクションからWebアプリにアクセス
  - トレーニング記録閲覧、体形変化閲覧、体重入力
  - LINEアプリ内ブラウザ or 外部ブラウザ (PWA) で利用
- **トレーナー**: iPad の Google カレンダーからアクセス
  - カレンダー上のお客様予定をタップ → Webアプリの該当会員ページが開く
  - 本日のAI提案メニュー、過去メニュー、体形変化写真を確認
  - セッション中の記録 (音声入力)、写真撮影・管理

## 3. 技術スタック

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | Next.js 15 (App Router) | RSC/Server Actions による高速開発 |
| UI | Tailwind CSS + shadcn/ui | モバイルファーストで高品質UI |
| バックエンド | Next.js API Routes + Server Actions | フロントと統合、別サーバー不要 |
| データベース | Supabase (PostgreSQL) | 認証・リアルタイム・ストレージ一体型 |
| 認証 | Supabase Auth | メール/パスワード認証 |
| 決済 | Stripe | サブスクリプション決済 |
| 音声文字起こし | OpenAI Whisper API | トレーナー音声→テキスト変換 |
| 写真ストレージ | Google Drive API | お客様ごとのフォルダ管理 |
| AI提案 | Gemini API | トレーニングデータ分析・メニュー提案 |
| 会員チャネル | LINE公式アカウント | 既存リッチメニューの1セクションからWebアプリへ誘導 |
| トレーナー導線 | Google Calendar (iPad) | カレンダーのイベントからWebアプリへ遷移 |
| デプロイ | Netlify | PWA対応 + ホスティング |
| 状態管理 | Zustand | 軽量でシンプル |

## 4. 業務フローとコア機能

### フロー1: 入会 (会員登録)

会員はLINE公式アカウントのリッチメニューから入会ページにアクセスし、手続きを完了する。

| 項目 | 詳細 |
|------|------|
| メールアドレス | ログインID として使用 |
| パスワード | Supabase Auth で管理 |
| 氏名 | 姓・名 |
| 電話番号 | 連絡用 |
| 生年月日 | 年齢に応じたメニュー考慮 |
| 利用規約同意 | チェックボックス + 規約全文表示 |
| Stripe 決済 | 月額サブスクリプション登録 |

**フロー**:
1. LINE リッチメニュー「入会はこちら」をタップ
2. Webアプリの入会ページが開く (LINEアプリ内ブラウザ)
3. 基本情報入力 (マルチステップフォーム)
4. 利用規約確認・同意
5. Stripe Checkout でカード登録・初回決済
6. アカウント作成完了 → 自動ログイン

**LINE リッチメニュー (既存)**:
- 既に運営中のリッチメニューがあり、その中の1セクションにPisteアプリのリンクを設置
- タップ → Webアプリの会員トップページ (入会 or マイページ) に遷移
- 既存メニューのレイアウト・他セクションには変更を加えない

### フロー2: カウンセリング

初回来店時にトレーナーがヒアリングし、アプリに記録する。

#### 2.1 性格診断 → 指導ポイント自動生成
- 選択式アンケート
- トレーニングスタイルの傾向把握 (ストイック / マイペース / 褒められたい等)
- AI提案のパーソナライズに活用
- **診断結果からAI (Gemini) が「指導ポイント」を自動生成**:
  - 性格タイプ名 (例: 「褒められ伸びる型」「ストイック追込み型」「コツコツ積上げ型」)
  - トレーナー向け指導ポイント 3〜5項目 (声かけの仕方、追い込み方、モチベ管理)
  - トレーナーが確認・編集可能
  - セッション画面 (`/trainer/members/[id]/today`) の冒頭に常時表示

#### 2.2 体のお悩み
- 部位別の悩み選択 (肩こり、腰痛、姿勢、etc.)
- 既往歴・ケガの有無
- 目標設定 (ダイエット、筋力UP、姿勢改善、etc.)

#### 2.3 食事
- 食事回数・時間帯
- 食事内容の傾向
- アレルギー・制限事項
- 改善目標

### フロー3: トレーニング (日々のセッション管理)

#### 3.0 トレーナーの導線: Google カレンダー → Webアプリ (iPad)

トレーナーはiPadのGoogleカレンダーを起点にセッションを運用する。

**カレンダー上のイベント構成**:
```
Google カレンダー イベント
├── タイトル: "田中太郎 様 14:00-15:00"
├── 説明欄:
│   ├── 🏋️ 本日のメニュー (リンク)
│   │   → https://piste-app.netlify.app/trainer/members/[id]/today
│   │     AI提案メニュー + 過去メニュー を1画面で表示
│   │
│   └── 📸 写真 (リンク)
│       → https://piste-app.netlify.app/trainer/members/[id]/photos
│         体形変化タイムライン表示
│
└── ※ イベント作成時に自動でリンク埋込 (Google Apps Script or 手動)
```

**トレーナーのセッション運用フロー**:
```
1. iPad で Google カレンダーを開く
2. 本日の予定から次のお客様をタップ
3. 「本日のメニュー」リンクをタップ
   → Webアプリが開く:
   ┌──────────────────────────┐
   │ 田中太郎 様 - 2026/03/19   │
   ├──────────────────────────┤
   │ 🧠 性格タイプ: 褒められ伸びる型 │
   │                           │
   │ 💡 指導ポイント:            │
   │  ・小さな成長も言葉で伝える    │
   │  ・重量UPは本人に気づかせる    │
   │  ・追い込みすぎず達成感重視    │
   │  ・「前回より良い」を具体的に   │
   ├──────────────────────────┤
   │ 🤖 AI提案メニュー          │
   │  ベンチプレス  60kg × 10 × 3 │
   │  スクワット    80kg × 8 × 3  │
   │  ...                      │
   │  [この提案を採用] [修正する]   │
   ├──────────────────────────┤
   │ 📋 過去のメニュー           │
   │  前回 (03/12): ベンチ 57.5kg │
   │  前々回 (03/05): ベンチ 55kg │
   │  ... (重量推移が一目でわかる)  │
   └──────────────────────────┘

4. 「写真」リンクをタップ
   → 体形変化タイムラインが開く:
   ┌──────────────────────────┐
   │ 田中太郎 様 - 体形変化      │
   ├──────────────────────────┤
   │ [入会時] → [1ヶ月] → [2ヶ月] │
   │  正面 / 側面 / 背面 切替     │
   │  [新しい写真を撮影]          │
   └──────────────────────────┘

5. セッション実施 → 音声録音 or テキストで記録
```

**Google カレンダーへのリンク自動埋込** (将来実装):
- 会員登録時にカレンダーイベントテンプレートを生成
- Google Calendar API or Apps Script でイベント説明欄にリンクを自動挿入
- 初期はトレーナーが手動でリンクを貼る運用でも可

#### 3.1 トレーナー側: セッション記録 (フォルダ監視 + 自動ワークフロー)

トレーナーがフォルダにファイルを置くだけで、自動的にワークフローが起動する。
音声ファイル・文字起こしテキストの両方に対応し、入力形式を自動判別する。

**監視フォルダ構成**:
```
Google Drive (or ローカル同期フォルダ)
└── Piste/
    └── inbox/
        └── [会員名_ID]/
            └── YYYY-MM-DD/
                ├── session_01.m4a      ← 音声ファイル (録音データ)
                ├── session_01.txt      ← または文字起こしテキスト
                └── session_02.m4a      ← 複数ファイル可
```

**フォルダ監視 Hook**:
- Supabase Storage の `on insert` トリガー、または Google Drive Webhook で監視
- ファイルが inbox に挿入されたことを検知 → ワークフロー起動

**自動ワークフロー (入力形式の自動判別)**:

```
ファイル検知 (Hook)
    │
    ├─ 拡張子判定
    │   ├─ 音声 (.m4a, .mp3, .wav, .webm)
    │   │   └─ Route A: 音声パイプライン
    │   │       1. Whisper API で文字起こし
    │   │       2. 文字起こし結果を保存 (.txt)
    │   │       3. → Route B へ合流
    │   │
    │   └─ テキスト (.txt, .md)
    │       └─ Route B: テキストパイプライン
    │           1. テキスト読み込み
    │           2. Gemini API で構造化データ抽出:
    │              - 種目名
    │              - 重量 (kg)
    │              - レップ数
    │              - セット数
    │              - トレーナーコメント
    │           3. 構造化データを DB に仮保存 (status: pending)
    │           4. トレーナーに通知 (確認待ち)
    │
    └─ トレーナーが確認・修正 → 確定 (status: confirmed)
```

**Route A (音声) のメリット**: トレーナーはセッション中にiPhoneで録音するだけでOK
**Route B (テキスト) のメリット**: 別アプリで文字起こし済みの場合、Whisper コストを省略可能

**ワークフロー実装方式の選択肢**:

| 方式 | 仕組み | 適用場面 |
|------|--------|----------|
| Supabase Storage トリガー | Storage に upload → DB function → Edge Function 呼出 | アプリ内アップロード |
| Google Drive Webhook | Drive の push notification → API Route で受信 | Drive に直接置く運用 |
| ローカル監視 (chokidar) | 同期フォルダの変更をNode.jsで検知 → API呼出 | 開発・テスト用 |

> **設計方針**: Hook のトリガー元は差し替え可能にし、ワークフロー本体 (判別→文字起こし→構造化→保存) は共通のパイプライン関数として実装する。

**写真管理フロー**:
1. トレーナーが iPhone でユーザーの体形写真を撮影
2. アプリから写真をアップロード
3. Google Drive にお客様ごとのフォルダを自動作成
4. 日付付きで写真を整理・保存
5. 体形変化の時系列管理 (before/after/経過)

**メニュー管理**:
- Google Drive 上のスプレッドシートと連携
- エクササイズ種目・セット数・重量を管理
- セッションごとのメニューを事前準備

#### 3.2 ユーザー側: 記録閲覧・入力

**閲覧機能**:
- トレーニング記録の閲覧 (日付別、種目別)
- 体形変化の写真閲覧 (Google Drive 連携)
- 重量推移グラフ

**入力機能**:
- 体重入力 (日々の自己記録)
- 体重推移グラフ表示

### フロー4: AI最適メニュー提案

トレーニングデータと体重データを元に、AIがトレーナーにメニューを提案する。

**入力データ**:
- 過去のトレーニング記録 (種目、重量、レップ数の推移)
- 体重推移データ
- カウンセリング情報 (目標、体の悩み、性格タイプ)

**出力**:
- 次回セッションの推奨メニュー
- 各種目の推奨重量 (過去データからの漸進的負荷)
- 注意点・コメント

**対象**: トレーナー向け (トレーナーが確認・調整してからセッションに使用)

## 6. 外部サービス連携

### 6.1 Stripe (決済)
- Checkout Session で初回カード登録
- Webhook で支払いステータス同期
- Customer Portal で会員自身がプラン管理
- サブスクリプションの自動課金

### 6.2 Google Drive (写真・メニュー管理)
- サービスアカウントによる API 接続
- 会員ごとにフォルダ自動作成 (`/Piste/[会員名]_[ID]/`)
- 撮影日付でサブフォルダ整理
- メニュー管理用スプレッドシート連携

### 6.3 OpenAI Whisper (音声文字起こし)
- トレーナーの音声録音をアップロード
- 日本語音声 → テキスト変換
- 変換後テキストを Gemini API で構造化

### 6.4 Gemini API (AI提案)
- トレーニングデータ + 体重 + カウンセリング情報を入力
- 次回メニュー・重量を提案
- トレーナーに提示 (最終判断はトレーナー)

### 6.5 LINE公式アカウント (会員フロントチャネル)
- 既存リッチメニューの1セクションにPisteアプリのリンクを設置
- LINEアプリ内ブラウザでWebアプリを開く (外部ブラウザへの切替も可)
- 既存の運用・他セクションには変更を加えない

### 6.6 Google Calendar (トレーナー導線)
- トレーナーはiPadでGoogleカレンダーを起点にセッション運用
- イベント説明欄にWebアプリへのリンクを埋込 (会員別ページ)
  - 「本日のメニュー」: `/trainer/members/[id]/today`
  - 「写真」: `/trainer/members/[id]/photos`
- 初期は手動リンク貼付、将来的にGoogle Calendar API or Apps Script で自動化

## 8. 開発フェーズ

### Step 1: プロジェクト初期化
- Next.js プロジェクト作成
- Tailwind CSS + shadcn/ui セットアップ
- Supabase プロジェクト作成・DB接続
- Netlify デプロイ設定 (netlify.toml)
- PWA 設定 (manifest.json, Service Worker, アイコン)
- 基本レイアウト (トレーナー用 / 会員用)

### Step 2: 入会フロー
- マルチステップ登録フォーム (Zodバリデーション)
- Supabase Auth 認証
- Stripe Checkout 統合
- Webhook でサブスクリプション同期
- 利用規約ページ

### Step 3: カウンセリング
- 性格診断アンケートUI
- 体のお悩み入力フォーム
- 食事ヒアリングフォーム
- カウンセリングデータ保存・閲覧

### Step 4: トレーニング記録パイプライン (フォルダ監視 + 自動処理)
- inbox フォルダ監視 Hook 実装 (Supabase Storage trigger / Google Drive Webhook)
- ファイル種別自動判別 (音声 or テキスト)
- Route A: 音声 → Whisper API 文字起こし → テキスト保存
- Route B: テキスト → Gemini API 構造化データ抽出 (種目・重量・レップ・セット)
- pipeline_jobs によるステータス管理 (queued → transcribing → structuring → pending_review)
- トレーナー確認・修正UI → 確定 (confirmed)
- 音声録音UI (アプリ内からの直接録音もサポート)

### Step 5: 写真管理
- カメラ撮影 / ファイル選択UI
- Google Drive API フォルダ自動作成
- 写真アップロード・整理
- 体形変化のタイムライン表示

### Step 6: 会員向け機能
- トレーニング記録閲覧
- 体形変化写真閲覧
- 体重入力・推移グラフ

### Step 7: AIメニュー提案
- トレーニングデータ集約
- Gemini API でメニュー提案生成
- トレーナー向け提案表示UI
- 提案の承認/修正フロー

### Step 8: 仕上げ
- レスポンシブ最適化 (モバイルファースト)
- エラーハンドリング
- パフォーマンス最適化
- テスト

## 9. 非機能要件

- **モバイルファースト**: トレーナーはiPhoneでセッション中に操作
- **パフォーマンス**: Core Web Vitals 良好スコア
- **セキュリティ**: RLS でデータアクセス制御、Stripe PCI-DSS 準拠
- **プライバシー**: 体形写真はGoogle Drive で安全に管理、アプリDBには参照IDのみ保持
- **可用性**: Netlify + Supabase のマネージドインフラ
- **PWA**: Service Worker によるオフライン対応、ホーム画面追加、プッシュ通知対応
